(inner) afficherFinDePartie()

Gère la fin de la partie en désactivant les options de vote et en affichant les options de clôture. Appelée lorsque toutes les missions ont été terminées ou lorsqu'une condition de fin de partie est atteinte. Les actions effectuées sont les suivantes : Désactive le bouton "Valider Vote" pour empêcher tout nouveau vote. Affiche les boutons "Suivant" et "Nouvelle Partie" pour permettre à l'utilisateur de naviguer ou de recommencer. Si toutes les missions sont terminées : Désactive le bouton "Suivant". Affiche le bouton "Télécharger Résultats" pour permettre de sauvegarder les résultats finaux dans un fichier JSON.

(inner) afficherTacheSuivante()

Gèrer la progression à travers les tâches du backlog. Incrémente l'indice du backlog (`backlogIndex`). Réinitialise les données nécessaires pour la nouvelle tâche : Efface les votes des joueurs. Réinitialise l'indice du joueur actuel et le compteur de tours. Met à jour l'interface a jour pour une nouvelle tache. Si toutes les tâches ont été complétées, la fonction : Affiche un message indiquant la fin des missions. Appelle `afficherFinDePartie()`.

(inner) demanderPause()

Cette fonction met la partie en pause si tous les joueurs ont sélectionné la carte "café". Elle demande une confirmation à l'utilisateur avant de sauvegarder la progression actuelle. Les étapes effectuées par cette fonction sont les suivantes : Affiche une boîte de dialogue de confirmation pour vérifier si l'utilisateur souhaite mettre la partie en pause. Si confirmé : Appelle la fonction `sauvegarderProgression` pour enregistrer l'état actuel du jeu. Avertit les utilisateurs qu'un fichier JSON contenant la progression a été téléchargé.

(inner) gererMoyenne(votesJoueurs, backlog, backlogIndex, demarrerMinuteur, afficherFinDePartie)

Cette fonction suit une logique en deux étapes pour calculer une note moyenne : **Premier tour :** Analyse les votes des joueurs pour identifier les valeurs minimale et maximale. Identifie les joueurs ayant donné les votes extrêmes. Affiche un message incitant les joueurs à discuter. Démarre un minuteur de 30 secondes permettant une réflexion avant de passer au second tour. Après le minuteur : Réinitialise les votes. Permet à tous les joueurs de revoter. //**Deuxième tour :** Calcule la moyenne des votes donnés par les joueurs. Enregistre la note moyenne associée à la tâche actuelle dans `resultatFinal`. Affiche un message avec la moyenne obtenue. Appelle `afficherFinDePartie()` pour conclure la tâche en cours.

(inner) gererUnanimite(votesJoueurs, backlog, backlogIndex, demarrerMinuteur, obtenirJoueurParVote)

Cette fonction vérifie si tous les votes des joueurs sont identiques pour établir l'unanimité : Si les votes diffèrent (absence d'unanimité) : Identifie les joueurs ayant voté pour les valeurs minimale et maximale. Affiche un message incitant les joueurs à discuter. Lance un minuteur de discussion (30 secondes) pour permettre une nouvelle phase de vote. Si les votes sont identiques (unanimité) : Enregistre la tâche actuelle avec la note unanime dans les résultats finaux. Affiche un message indiquant que l'unanimité est atteinte. Appelle `afficherFinDePartie()` pour conclure la tâche en cours.

(inner) sauvegarderProgression()

Cette fonction sauvegarde la progression actuelle du jeu dans un fichier JSON. Le fichier est automatiquement téléchargé par le navigateur. Elle crée un objet contenant les données de la progression actuelle. Convertit cet objet en une chaîne JSON formatée. Crée un fichier temporaire (Blob) contenant cette chaîne JSON. Génère un lien temporaire pour permettre le téléchargement du fichier. Déclenche automatiquement le téléchargement du fichier nommé `progression.json`.

(inner) sauvegarderResultats(resultatFinal)

Cette fonction sauvegarde les résultats finaux du jeu dans un fichier JSON et déclenche automatiquement le téléchargement de ce fichier. Les étapes effectuées par cette fonction sont les suivantes : Convertit l'objet `resultatFinal` (données des résultats) en une chaîne JSON bien formatée. Crée un fichier temporaire (Blob) contenant cette chaîne JSON. Génère une URL temporaire pointant vers ce fichier Blob. Simule un clic sur un lien de téléchargement, ce qui permet de télécharger le fichier. Libère les ressources associées à l'URL temporaire une fois le fichier téléchargé.

btnChargerBacklog

Gère le chargement d'un fichier JSON pour le backlog. Cette fonction est exécutée lorsqu'un utilisateur clique sur le bouton "Charger Backlog". Elle effectue les actions suivantes : Vérifie si un fichier a été sélectionné. Lit le fichier JSON en utilisant un `FileReader`. Valide que le contenu du fichier est un tableau non vide. Met à jour la variable globale `backlog` avec les données du fichier. Affiche des messages d'erreur ou de succès selon le cas. * @throws {Error} Si le fichier JSON est vide ou mal formaté.

btnDemarrer

Initialisation d'une nouvelle partie. Evenement est exécutée lorsqu'un utilisateur clique sur le bouton "Démarrer". Les actions effectuées sont les suivantes : Récupère les pseudos des joueurs depuis les champs d'entrée. Lit le mode de jeu sélectionné dans la liste déroulante. Sinon, démarre la partie et configure l'interface utilisateur.

btnValiderVote

Lorsque l'on clique sur le bouton valider le vote : Vérifie si le joueur actuel a soumis un vote valide. Si aucun vote n'est trouvé, affiche un message d'erreur. Si le vote est valide : Passe au joueur suivant en incrémentant `joueurActuelIndex`. Si tous les joueurs ont voté : Traite les votes en fonction du mode de jeu : **Mode "strict"** : Appelle `gererUnanimite()` pour vérifier l'unanimité. **Mode "moyenne"** : Appelle `gererMoyenne()` pour calculer la moyenne.
